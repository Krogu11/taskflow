# Hiring Plan — Founding Engineer

## Priority: CTO / Founding Engineer (Hire #1)

### Why first
Technical leadership is the critical path constraint. Without a founding engineer,
the product can't be built. Every other hire depends on having infra, a codebase,
and a working dev loop in place.

### Role Definition

**Title:** CTO / Founding Engineer  
**Reports to:** CEO  
**Type:** AI Agent (claude_local adapter)

**Core responsibilities:**
- Own the full technical stack: architecture, implementation, deployment
- Set up dev infrastructure (CI/CD, environments, tooling)
- Implement product features as directed by the CEO
- Maintain code quality and technical velocity
- Escalate blockers and surface trade-offs to the CEO

**What we're NOT asking for:**
- Product strategy (CEO owns this)
- Hiring decisions (CEO-led until Series A)
- Customer-facing communication

### Hire #2 and beyond (post-CTO)
Once the CTO is operational and the codebase is established:

1. **CMO** — when we need to grow beyond word-of-mouth
2. **UX Designer** — when user-facing product quality becomes the constraint
3. **Additional engineers** — when CTO is bottlenecked on implementation

---

## Roadmap: Concrete tasks for the CTO

1. Choose tech stack and set up repository + CI/CD pipeline
2. Implement authentication and user management
3. Build core product MVP (to be refined with CEO)
4. Set up observability (logging, error tracking, monitoring)
5. Deploy to production environment

---

## Delegation model
- CEO sets priorities weekly via issues
- CTO owns all technical subtasks
- CEO reviews and unblocks on request
- Escalation path: CTO → CEO → Board (Fabian)
